## Требования к безопасности
1. Собственники проходят аутентификацию и авторизацию для доступа к функциям домофона и шлагбаума.
2. Собственники предоствляют согласие на обработку персональных данных при вводе персональных данных, включаяя биометрию.
3. Собственники имеют доступ только к собсвенным персональным даннным.
4. Собсвенники предоствляют согласие на обработку персональных данных внешнему поставщику для подключения функций домофона и шлагбаума. 
5. Персональные даннные Собственников, включая биометрию, хранятся в зашифрованном виде как внутрри системы, так и на внешней платформе.
6. Коммуникации Собственников с системой и системы с внешней платформой защищены шифрованием с применением протокола TLS 1.3+.
7. При расторжении договора персональные данные Собственника удаляются из системы и с внешней платформы.
8. Для входящего и исходящего трафика с внешней платформой используется межсетевой экран (firewall).
9. Осуществляется журналирование доступа к персональным данным.
10. Осуществляется журналирование входящих/исходящих запросов из/во внешнюю платформу.
## Протоколы аутентификации и авторизации
1. Для аутентификации Собственник вводит логин и пароль. Используется существующее решение на базе Keycloack.
2. В качестве протокола авторизации в системе используется OAuth 2.0.
3. Атентификации между системой и внешней платформой осуществляется посредством mTLS.  
4. Для аутентификации и авторизации Собственников на внешней платформе передается JSON Web Token (JWT).  
## Описание взаимодействия между системами предприятия и внешней платформой
### Интеллектуальный домофон
#### Подключениее услуги распознования по биометрии.

- Собсвенник решает подключить распознование по биометрии и нажимает соответствующую кнопку в мобильном приложении.
- Мобильное приложение посылает запрос на сервис tenant-core-app через rest-api.
- Сервис tenant-core-app получает биометрию из сервиса CRM через rest-api.
- Сервис tenant-core-app отправляет запрос на подключение с биометрией на внешнюю платфоррму через rest-api по протоколу https.
#### Открытие двери

- С внешней платфоррмы приходит вызов на SIP server.
- SIP server определяет мобильное устройство Собственника и осуществляеет вызов через push-уведомление.
- Мобильное устройство запускает приложение и устанавливает соединение с домофоном через механизм SIP.
- Видеопоток с камеры домофона передается через RTP Proxy.
- Собственник видит лицо гостя на экране мобильного устройства и решает, разрешить доступ в здание или отклонить запрос.
- Собственник решает открыть дверь и нажимаеет кнопку в приложении.
- Приложение отпправляет запрос "Открыть" на внешнюю платформу через механизм SIP.
### Интеллектуальный шлагбаум
#### Подключениее услуги автоматического открытия по номеру автомобиля

- Собсвенник решает подключить услугу автоматического открытия по номеру автомобиля и нажимает соответствующую кнопку в мобильном приложении.
- Мобильное приложение посылает запрос на сервис tenant-core-app через rest-api.
- Сервис tenant-core-app получает номер автомобиля из сервиса CRM через rest-api.
- Сервис tenant-core-app отправляет запрос на подключение с номером на внешнюю платфоррму через rest-api по протоколу https.
#### Открытие шлагбаума

- Собственник решает открыть шлагбаум и нажимает соответсвующую кнопку в мобильном приложении.
- Мобильное приложение посылает запрос на внешнюю платформу через rest-api по протоколу https.
